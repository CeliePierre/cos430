1. The UML for a class has 3 sections. What feature of the class goes
   in each section. 
The class header, the attributes (properties), and the operations (methods). 
The class header includes the class name and any inheritance or implementation information. 
The attributes section lists the properties of the class, 
including the property name, type, and visibility. 
The operations section lists the methods of the class, including the method name, parameters, and return type.

2. Syntax is always expressed in general terms.  The syntax for the 
   declaration of a variable is:       dataType identifier;
   Looking at the UML, what is the UML syntax for a property?

The UML syntax for a property is usually shown as an attribute with its name, type, and visibility. The visibility can be represented by + (public), - (private), # (protected), or (package) before the name of the attribute.For example: name: String

========================================================================
3. What is the UML syntax for a method?
The UML syntax for a method is usually shown as an operation with its name, parameters, and return type. The visibility can be represented by + (public), - (private), # (protected), or (package) before the name of the operation. For example:
getName(): String
